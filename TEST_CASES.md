## 1. 配置文件测试

### 1.1 配置文件加载

#### TC-CONF-001: 加载有效配置文件
**优先级**: 高  
**类型**: 正向测试  
**前置条件**: 配置文件存在且格式正确  

**输入**:
```yaml
region: us-west-2
access_key: test_access_key
secret_key: test_secret_key
endpoint: http://localhost:9000
force_path_style: true
use_accelerate: false
```

**预期结果**:
- 配置文件成功加载
- 所有配置项正确解析
- GlobalConfig 被正确设置

#### TC-CONF-002: 配置文件不存在
**优先级**: 高  
**类型**: 异常测试  
**前置条件**: 配置文件路径不存在  

**输入**: `/nonexistent/path/config.yaml`

**预期结果**:
- 返回错误
- 错误信息包含文件不存在

#### TC-CONF-003: 无效的YAML格式
**优先级**: 高  
**类型**: 异常测试  

**输入**:
```yaml
region: us-west-2
access_key: [invalid: yaml: format
```

**预期结果**:
- 返回错误
- 错误信息包含 YAML 解析错误

#### TC-CONF-004: 空配置文件
**优先级**: 中  
**类型**: 边界测试  

**输入**: 空文件

**预期结果**:
- 成功加载（无错误）
- 使用默认值

### 1.2 环境变量覆盖

#### TC-CONF-005: 环境变量覆盖配置
**优先级**: 高  
**类型**: 正向测试  

**环境变量**:
```bash
AWS_REGION=ap-northeast-1
AWS_ACCESS_KEY_ID=env_access_key
AWS_SECRET_ACCESS_KEY=env_secret_key
AWS_ENDPOINT=http://env-endpoint:9000
```

**配置文件**:
```yaml
region: us-east-1
access_key: config_access_key
secret_key: config_secret_key
```

**预期结果**:
- 环境变量值覆盖配置文件值
- Region = ap-northeast-1
- AccessKey = env_access_key

### 1.3 默认值测试

#### TC-CONF-006: 配置默认值
**优先级**: 中  
**类型**: 边界测试  

**输入**:
```yaml
access_key: test_key
secret_key: test_secret
```

**预期结果**:
- Region = us-east-1（默认）
- ForcePathStyle = false（默认）
- UseAccelerate = false（默认）
