## 6. 测试执行要求

### 6.1 测试环境检查清单

- [ ] Go 1.21+ 已安装
- [ ] 配置文件 `~/.s3tool.yaml` 已正确配置
- [ ] S3端点网络可达
- [ ] 测试账号有足够的权限（创建、删除、读写）
- [ ] 没有其他测试正在并行执行

### 6.2 测试执行顺序建议

1. **单元测试先行**：验证基础逻辑（`go test ./config/... ./cmd/...`）
2. **连接性验证**：执行 `s3tool bucket list` 确认配置正常
3. **集成测试分组**：按模块执行，先核心后高级
4. **清理测试数据**：每个测试用例执行后清理测试对象
5. **最终验证**：执行完整E2E场景
